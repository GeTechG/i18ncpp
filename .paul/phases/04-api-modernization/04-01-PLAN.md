<objective>
## Goal
Modernize the i18ncpp public API with std::span parameters and simplified overload sets for tr/trPlural.

## Purpose
Reduces API surface (10 overloads → 6), enables zero-copy parameter passing from any contiguous container (vector, array, C-array), and completes the C++20 modernization milestone.

## Output
- Updated `include/i18ncpp.h` with std::span-based signatures
- Updated `src/i18ncpp.cpp` with span-based implementations
- Updated tests and benchmarks for the simplified API
</objective>

<task type="auto">
  <name>Task 1: Adopt std::span in header and implementation</name>
  <files>include/i18ncpp.h, src/i18ncpp.cpp</files>
  <action>
    **Header (i18ncpp.h):**
    1. Add `#include <span>` to includes
    2. Replace `tr(key, const std::vector<std::string>& params)` with `tr(key, std::span<const std::string> params)`
    3. Replace `trPlural(key, count, const std::vector<std::string>& params)` with `trPlural(key, count, std::span<const std::string> params)`
    4. Keep `tr(key, std::initializer_list<std::string>)` — needed for brace-init syntax `tr("k", {"a", "b"})`
    5. Keep `trPlural(key, count, std::initializer_list<std::string>)` — same reason
    6. Remove: `tr(key, const string& p1)`, `tr(key, p1, p2)`, `tr(key, p1, p2, p3)`
    7. Remove: `trPlural(key, count, p1)`, `trPlural(key, count, p1, p2)`
    8. Keep `trv`/`trPluralv` variadic templates — update internal call from vector to match span overload
    9. Update private `interpolateArray` signature to take `std::span<const std::string>`

    **Implementation (i18ncpp.cpp):**
    1. Update `tr(key, span)` core implementation — span is iterable, size() works, minimal changes
    2. Update `trPlural(key, count, span)` core — `extendedParamsBuf_` pattern still works (build vector, then pass as span to interpolateArray)
    3. Update `interpolateArray` to accept `std::span<const std::string>`
    4. Update `initializer_list` overloads to construct a vector and delegate to span overload
    5. Remove implementations of deleted individual-param overloads

    **Avoid:** Changing `translate(key, json)` — that uses named params via json, completely separate path.
    **Avoid:** Changing internal buffer pooling or cache key logic — only signatures change.
  </action>
  <verify>Build compiles: `cd build && cmake --build . 2>&1 | tail -5`</verify>
  <done>AC-1 and AC-2 satisfied: span-based API compiles, individual overloads removed</done>
</task>

  <files>tests/test_translation.cpp, tests/test_interpolation.cpp, tests/test_plural.cpp, benchmarks/bench_interpolation.cpp</files>
  <action>
    Find all call sites that used removed individual-param overloads and update them:
    - `tr("key", "p1")` → `trv("key", "p1")` or `tr("key", {"p1"})`
    - `tr("key", "p1", "p2")` → `trv("key", "p1", "p2")` or `tr("key", {"p1", "p2"})`
    - `tr("key", "p1", "p2", "p3")` → `trv("key", "p1", "p2", "p3")` or `tr("key", {"p1", "p2", "p3"})`
    - `trPlural("key", N, "p1")` → `trPluralv("key", N, "p1")` or `trPlural("key", N, {"p1"})`
    - `trPlural("key", N, "p1", "p2")` → `trPluralv("key", N, "p1", "p2")` or `trPlural("key", N, {"p1", "p2"})`

    Prefer `initializer_list` syntax `{"p1", "p2"}` for literal strings (more readable).
    Prefer `trv`/`trPluralv` when params come from variables.

    Check benchmarks in bench_interpolation.cpp for the same patterns.
  </action>
