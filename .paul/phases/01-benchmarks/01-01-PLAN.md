<objective>
## Goal
Create a benchmark suite measuring baseline performance of all hot paths: key lookup (tr), pluralization (trPlural), interpolation, and formatting (formatNumber, formatPrice, formatDate). Results saved to file for comparison after optimization phases.

## Purpose
Without measured baselines, we cannot quantify the impact of optimizations in Phases 2-5. This suite becomes the "before" snapshot.

## Output
- `benchmarks/` directory with micro-benchmark files
- CMake integration via `BUILD_BENCHMARKS` option
- Baseline results captured in `benchmarks/BASELINE.md`
</objective>

<task type="auto">
  <name>Task 1: Create benchmark infrastructure</name>
  <files>CMakeLists.txt, benchmarks/CMakeLists.txt</files>
  <action>
    1. Add BUILD_BENCHMARKS option to root CMakeLists.txt (same pattern as BUILD_TESTS)
    2. Create benchmarks/CMakeLists.txt:
       - Use a simple custom micro-benchmark harness (no external dependency) — just a header with a run_benchmark() function that measures N iterations with chrono::high_resolution_clock
       - Create benchmarks/bench_harness.h with:
         - run_benchmark(name, iterations, lambda) that prints ns/op
         - Warm-up pass (10% of iterations, discarded)
         - Default iterations: 100000
       - Add benchmark executable target linked to i18ncpp and nlohmann_json
       - Pass FIXTURES_DIR like tests do
    Avoid: Google Benchmark dependency — keep it dependency-free with a simple chrono-based harness
  </action>
  <verify>cmake -DBUILD_BENCHMARKS=ON .. configures without error</verify>
  <done>AC-1 satisfied: benchmark target builds successfully</done>
</task>

<task type="auto">
  <name>Task 2: Write benchmarks for all hot paths</name>
  <files>benchmarks/bench_lookup.cpp, benchmarks/bench_interpolation.cpp, benchmarks/bench_formatting.cpp</files>
  <action>
    Create three benchmark files, each with main() using bench_harness.h:

    bench_lookup.cpp:
    - BM_TrSimpleKey: tr("greeting") — simple flat lookup
    - BM_TrNestedKey: tr("nested.deep.key") — dot-notation key
    - BM_TrMissingFallback: tr with missing key falling back to fallback locale
    - BM_TrPlural: trPlural("items", count) — plural form selection
    - BM_KeyExists: keyExists("greeting") — existence check

    bench_interpolation.cpp:
    - BM_InterpolatePositional: tr("msg", "arg1", "arg2") — positional params
    - BM_InterpolateNamed: translate("msg", {{"name", "value"}}) — named params
    - BM_InterpolateMixed: multiple interpolation patterns in one string

    bench_formatting.cpp:
    - BM_FormatNumber: formatNumber(123456.78)
    - BM_FormatPrice: formatPrice(99.99)
    - BM_FormatDate: formatDate("short_date", &tm) — with a fixed tm struct
    - BM_FormatNumberCached: second call to formatNumber (cache hit)

    Each file: load en.json fixture in setup, run benchmarks, print results table.
  </action>
  <verify>Build and run each benchmark executable — all print ns/op results without crashes</verify>
  <done>AC-2 satisfied: at least 6 distinct benchmarks covering all hot paths execute and report timing</done>
</task>
