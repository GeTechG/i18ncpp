<objective>
## Goal
Add translation result caching for tr/trPlural to skip regex interpolation on repeated calls, and provide a constexpr FNV-1a hash utility for compile-time key hashing.

## Purpose
Regex-based interpolation is the #1 bottleneck (1545-6775 ns/op). Caching final results at the tr/trPlural level eliminates regex cost entirely on cache hits. Constexpr hashing enables zero-runtime-cost key hashing for statically known translation keys.

## Output
- Translation result cache in I18N class with automatic invalidation
- Constexpr FNV-1a hash function usable at compile time
- All 82 existing tests passing
</objective>

    1. In i18ncpp.h, add mutable member:
       `mutable std::unordered_map<std::string, std::string> translationCache_;`
    2. Add public method: `size_t translationCacheSize() const noexcept;`
    3. Add private method: `void clearTranslationCache();`

    4. In i18ncpp.cpp, implement caching in `tr(key, vector<string>& params)`:
       - Build cache key: key + "\0" + params[0] + "\0" + params[1] + ... (use null byte separator — cannot appear in user strings)
       - Check translationCache_ for hit; return immediately if found
       - On miss, compute result via existing logic, store in cache, return

    5. Implement caching in `trPlural(key, count, vector<string>& params)`:
       - Build cache key: key + "\0P\0" + to_string(count) + "\0" + params...
       - Same hit/miss pattern as tr

    6. Call `clearTranslationCache()` in all mutation paths:
       - setLocale (both overloads), setFallbackLocale, load, loadLocale, mergeLocale, mergeLocaleFromFile, configure, reset
       - Note: these already call clearFormatCache() — add clearTranslationCache() alongside

    7. In reset(), also clear translationCache_

    Avoid: Caching translate() (the json-params overload) — json param hashing is expensive and translate() is the less-used API path.
    Avoid: Complex LRU eviction — unbounded cache is fine for i18n (translation sets are finite and small).

<task type="auto">
  <name>Task 2: Add constexpr FNV-1a hash utility</name>
  <files>include/i18ncpp.h</files>
  <action>
    1. Add constexpr FNV-1a hash function in the i18n namespace (before the I18N class):
       ```
       constexpr std::size_t hash(const char* str, std::size_t basis = 14695981039346656037ULL) noexcept {
           return (*str == '\0') ? basis : hash(str + 1, (basis ^ static_cast<std::size_t>(*str)) * 1099511628211ULL);
       }
       ```
    2. Add a string_view overload using a loop (constexpr in C++17):
       ```
       constexpr std::size_t hash(std::string_view sv) noexcept {
           std::size_t h = 14695981039346656037ULL;
           for (char c : sv)
               h = (h ^ static_cast<std::size_t>(c)) * 1099511628211ULL;
           return h;
       }
       ```

    This provides users with compile-time key hashing for future use (e.g., pre-hashed lookup tables).
    The function is header-only and zero-cost when used with literals.

    Avoid: Changing the internal map structure — transparent hash is deferred to C++20.
    Avoid: Adding macros — constexpr functions are sufficient in C++17.
  </action>
  <verify>Add static_assert in the header or a test: `static_assert(i18n::hash("test") != 0)` to prove compile-time evaluation.</verify>
