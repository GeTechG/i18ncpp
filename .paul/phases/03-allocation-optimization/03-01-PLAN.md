<objective>
## Goal
Reduce heap allocations in interpolation and plural-lookup hot paths by reusing mutable member buffers and fixing O(n^2) vector construction.

## Purpose
Interpolation is the #1 bottleneck (~6775 ns/op named, ~1545 ns/op positional). Each call allocates multiple temporary strings and vectors. By pooling these buffers as mutable members and fixing the quadratic `argsToStrings`, we reduce per-call allocation pressure without changing any public API.

## Output
Modified `include/i18ncpp.h` and `src/i18ncpp.cpp` with reusable interpolation buffers, optimized `argsToStrings`, and pooled `extendedParams` in `trPlural`.
</objective>

<task type="auto">
  <name>Task 2: Fix O(n^2) argsToStrings with parameter pack expansion</name>
  <files>include/i18ncpp.h</files>
  <action>
    Replace the recursive argsToStrings template with a single-pass implementation:

    ```cpp
    template<typename... Args>
    std::vector<std::string> argsToStrings(const Args&... args) const {
        std::vector<std::string> result;
        result.reserve(sizeof...(args));
        (result.push_back(toString(args)), ...);
        return result;
    }
    ```

    This uses C++17 fold expression to expand all args in order with push_back (O(1) amortized each) instead of the current recursive approach that calls insert(begin) which is O(n) per element.

    Remove the two recursive template overloads (single-arg and multi-arg versions).
    Keep the zero-arg overload.

    Avoid: Changing the toString helper. Avoid changing trv/trPluralv signatures.
  </action>
  <verify>Build succeeds; run test suite — all 82 tests pass; specifically verify trv and trPluralv work correctly with 1, 2, and 3 args</verify>
  <done>AC-2 satisfied: argsToStrings builds vector in single O(n) pass via fold expression</done>
</task>
