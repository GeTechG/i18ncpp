## AC-1: load(json) replaces locale data
```gherkin
Given a locale "en" is loaded with keys {greeting: "hello", farewell: "bye"}
When load(json) is called again with "en" containing only {greeting: "hi"}

```

## AC-2: Format config resets on locale switch
```gherkin
Given locale "de" has _formats with decimal_symbol=","
And locale "en" has no _formats
When setLocale("de") then setLocale("en")
Then defaultConfig uses the baseline default decimal_symbol="." (not ",")
```

## AC-3: trv/trPluralv serialize bool as "true"/"false"
```gherkin
Given a translation key with placeholder {1}
When trv(key, true) is called
Then the interpolated value is "true" (not "1")
And trv(key, false) produces "false" (not "0")
```

## AC-4: keyExists resolves plural/variant suffixes and fallback
```gherkin
Given locale "en" has key "items.one" and "items.other"
When keyExists("items") is called
Then it returns true (matching tr/trPlural resolution behavior)
```

<task type="auto">
  <name>Task 2: Fix format config isolation with baselineConfig_ (F2)</name>
  <files>src/i18ncpp.cpp, include/i18ncpp.h</files>
  <action>
    The bug: configure() mutates defaultConfig, formatConfigs stores it per-locale,
    but setLocale() only restores from formatConfigs — if the new locale has no entry,
    the previous locale's config leaks.

    Fix:
    1. In include/i18ncpp.h, add a new private member `FormatConfig baselineConfig_;`
       right after `FormatConfig defaultConfig;` — this stores the pristine default.
    2. In I18N constructor (src/i18ncpp.cpp), baselineConfig_ is default-constructed
       (same as defaultConfig), so no constructor change needed.
    3. In I18N::reset(), also reset baselineConfig_ to default: `baselineConfig_ = FormatConfig{};`
    4. In setLocale(string_view) at line ~155: when formatConfigs lookup misses,
       reset defaultConfig to baselineConfig_ instead of leaving it stale:
       ```
       auto it = formatConfigs.find(locales[0]);
       if (it != formatConfigs.end()) {
           defaultConfig = it->second;
       } else {
           defaultConfig = baselineConfig_;
       }
       ```
    5. Same pattern in setLocale(vector) at line ~168.
    6. In configure(), after committing to defaultConfig via copy-and-swap,
       do NOT update baselineConfig_ — it stays pristine. configure() only affects
       the current runtime config + the per-locale entry in formatConfigs.
  </action>
  <verify>New test test_format_isolation passes — switching to locale without _formats resets to baseline</verify>
  <done>AC-2 satisfied: format config resets on locale switch</done>
</task>

<task type="auto">
  <name>Task 3: Fix bool branch order in toString (F3)</name>
  <files>include/i18ncpp.h</files>
  <action>
    In the toString() template (line ~259), std::is_arithmetic_v<T> matches bool before
    the dedicated bool branch because bool is arithmetic in C++. Move the bool check
    above the arithmetic check:

    ```cpp
    template<typename T>
    std::string toString(const T& value) const {
        if constexpr (std::is_same_v<T, std::string> || std::is_same_v<T, std::string_view> || std::is_same_v<T, const char*>) {
            return std::string(value);
        } else if constexpr (std::is_same_v<T, bool>) {
            return value ? "true" : "false";
        } else if constexpr (std::is_arithmetic_v<T>) {
            return std::to_string(value);
        } else {
            std::ostringstream oss;
            oss << value;
            return oss.str();
        }
    }
    ```
  </action>
  <verify>New test test_bool_conversion passes — trv with bool produces "true"/"false"</verify>
  <done>AC-3 satisfied: bool serializes as "true"/"false" in trv/trPluralv</done>
</task>

<task type="auto">
  <name>Task 4: Fix keyExists to match tr/trPlural resolution (F4)</name>
  <files>src/i18ncpp.cpp</files>
  <action>
    Current keyExists (line ~824) only checks exact flat key via getTranslationData(key, loc).
    tr() also tries key.other; trPlural() tries key.{pluralForm}, key.other, key.{count}.

    Extend keyExists to also check the .other suffix (matching tr's fallback):
    - After the exact-key check, also try key + ".other" in each fallback locale.
    This makes keyExists("items") return true when "items.other" exists,
    matching what tr() would resolve.

    Do NOT replicate the full trPlural resolution (plural forms, count suffixes) —
    keyExists is meant to answer "will tr() find something?" and tr() uses .other
    as its fallback. This keeps the fix minimal and correct.
  </action>
