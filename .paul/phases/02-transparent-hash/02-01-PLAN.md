<objective>
## Goal
Enable heterogeneous (transparent) lookup on all unordered_maps so that string_view keys can query maps without allocating temporary std::string objects.

## Purpose
The hot path (getTranslationData) currently converts string_view to string for every map lookup — two temporary allocations per call. With C++20 transparent hash, lookups use string_view directly, eliminating these allocations. This was deferred from v0.3 because C++17 lacked support.

## Output
- All 4 unordered_map types use transparent hash/equal
- getTranslationData performs zero-allocation lookups
- Cache lookups (formatCache_, translationCache_) perform zero-allocation lookups
- All 82 tests pass, all benchmarks compile
</objective>

  <files>include/i18ncpp.h</files>
  <action>
    Add StringHash and StringEqual structs in the i18n namespace, before the config structs:

    StringHash:
    - Tag: using is_transparent = void;
    - operator()(std::string_view sv) — hash via std::hash<std::string_view>
    - operator()(const std::string& s) — hash via std::hash<std::string_view> on string_view of s
    - Both return size_t, both are const noexcept

    StringEqual:
    - Tag: using is_transparent = void;
    - operator()(std::string_view a, std::string_view b) — returns a == b
