<objective>
## Goal
Replace all 4 `std::regex` patterns in interpolation with hand-written parsers to eliminate the regex bottleneck. InterpolateNamed should drop from ~6740 ns/op to sub-200 ns/op. All positional/unnamed patterns also replaced for consistency and to remove the `<regex>` dependency entirely.

## Purpose
Named interpolation is the last remaining hot spot in i18ncpp. `std::regex` is orders of magnitude slower than manual string scanning for these simple patterns. Removing it completes the v0.3 performance story.

## Output
- Modified `include/i18ncpp.h` — remove regex members, remove `#include <regex>`
- Modified `src/i18ncpp.cpp` — replace regex_search loops with manual parsing
</objective>

<task type="auto">
  <name>Task 1: Replace regex patterns with manual parsers</name>
  <files>include/i18ncpp.h, src/i18ncpp.cpp</files>
  <action>
    Replace all 4 regex-based search loops with manual character-by-character parsing:

    **In `interpolate()` — replace `fieldPattern` regex (`%{key}`):**
    - Scan for `%{` literal (where preceding char is not `%`)
    - Find closing `}`
    - Extract key between `{` and `}`
    - Key chars: `[\w\.]` → alphanumeric, underscore, dot
    - Keep the same buffer append logic (interpolateBuf_)

    **In `interpolate()` — replace `valuePattern` regex (`%<key>.fmt`):**
    - Scan for `%<` literal (where preceding char is not `%`)
    - Find closing `>`
    - Expect `.` followed by single format char after `>`
    - Extract key and format char
    - Keep the same buffer append logic (interpolateBuf2_)

    **In `interpolateArray()` — replace `indexPattern` regex (`{N}`):**
    - Scan for `{` literal
    - Check if contents are all digits
    - Find closing `}`
    - Extract number, convert with simple loop (not stoi)
    - Keep the same buffer append logic

    **In `interpolateArray()` — replace `emptyBracePattern` regex (`{}`):**
    - Scan for literal `{}` two-char sequence
    - Simplest replacement — just find_str

    **In `i18ncpp.h`:**
    - Remove `#include <regex>`
    - Remove the 4 static `std::regex` member declarations
    - Remove the 4 static `const std::regex` definitions from i18ncpp.cpp

    **Important:**
    - Preserve exact same behavior for edge cases: `%%{key}` should NOT interpolate (escaped percent)
    - Preserve the "previous character" capture group behavior — the regex `([^%]|^)` captures one char before `%{` to handle escape detection
    - The manual parser must handle: start-of-string (no prev char), `%%` escape, and normal `%{` match identically to the regex
  </action>
