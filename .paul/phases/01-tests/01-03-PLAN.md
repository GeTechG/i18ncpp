<acceptance_criteria>

## AC-1: Number formatting with default config
```gherkin
Given a default-configured I18N instance
When formatNumber is called with 1234567.89
Then result uses default config: space thousand separator, dot decimal, 2 fract digits → "1 234 567.89"
```

## AC-2: Number formatting with custom config
```gherkin
Given configure() called with custom number config (comma thousand sep, dot decimal, 3 fract digits)
When formatNumber is called with 1234.5
Then result uses custom config → "1,234.500"
```

## AC-3: Negative number formatting
```gherkin
Given a default-configured I18N instance
When formatNumber is called with -42.5
Then result is "-42.50"
```

## AC-4: Currency formatting
```gherkin
Given configure() called with currency config (symbol="$", format="%c%p%q")
When formatPrice is called with 1234.56
Then result is "$1 234.56" (or per configured separators)
```

## AC-5: Date formatting with explicit tm and named patterns
```gherkin
Given a known std::tm (2026-03-15 14:30:45, Saturday)
When formatDate is called with "short_date", "long_date", "short_time", "long_time", custom pattern "%Y-%m-%d"
Then each returns correctly formatted string using the date config specifiers
```

## AC-6: Date formatting 12-hour and AM/PM
```gherkin
Given a known std::tm with hour=14
When formatDate is called with "%I:%M %p"
Then result is "02:30 PM"
```

## AC-7: configure() applies format settings
```gherkin
Given configure() called with JSON containing number, currency, and date_time sections
When getConfig() is inspected
Then all configured fields match the JSON input
```

## AC-8: mergeLocale adds translations to existing locale
```gherkin
Given locale "en" loaded from en.json
When mergeLocale is called with "en" and a file containing new keys
Then new keys are accessible via tr() and old keys still work
```

## AC-9: mergeLocaleFromFile extracts locale from filename
```gherkin
Given no locale explicitly specified
When mergeLocaleFromFile("path/to/en.json") is called
Then locale "en" is populated from the file
```

## AC-10: reset() clears all state
```gherkin
Given locales loaded and config customized
When reset() is called
Then getLocale() is empty, translations are gone, formatConfig is back to defaults
```

       - tm.tm_wday = 6 (Saturday), tm.tm_mon = 2 (March), tm.tm_year = 126

       - formatDate("%a %b %d", &tm) → "Sat Mar 15"
       - formatDate("%l, %F %d", &tm) → "Saturday, March 15"
       - formatDate("", &tm) → ISO format "2026-03-15T14:30:45"
