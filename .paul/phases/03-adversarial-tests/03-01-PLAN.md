<objective>
## Goal
Close AEGIS P-005 (F-TEST-003) by installing throw-site test coverage for `I18NError`. The current suite has **zero** `EXPECT_THROW` assertions — the public exception contract has no test evidence. This plan adds 5 hostile-input tests covering missing file, malformed JSON, top-level non-object, non-string leaf (silent-skip contract), and `_formats` type error (M1 regression coupling).

## Purpose
Phase 1 hardened the JSON/filesystem trust boundaries and Phase 2 enforced the cache-clear invariant by test enumeration. Both phases produced code that is *reasoned* to be correct — but the error paths have never been exercised by a test. F-TEST-003 flagged this as a structurally trivial gap: `grep EXPECT_THROW tests/` returns zero.

Without throw-site tests:
- A refactor that changes the exception type (e.g., leaks `json::type_error` past the `I18NError` boundary) is caught only by the one caller that wrote `catch(I18NError&)` and crashed.
- A missing catch block (such as the original `load(const json&)` before Phase 1) ships as a contract violation with no test signal.
- `I18NError`'s declaration at `include/i18ncpp.h:45` has no test that proves it's the thing thrown.

The exception contract is half the API. This plan tests the other half.

## Output
- New `tests/fixtures/hostile/` directory with 3 hostile JSON fixtures (`not_an_object.json`, `malformed_json.json`, `leaf_is_int.json`).
- New `tests/test_hostile_input.cpp` with 5 test cases covering all throw-site categories plus the silent-skip contract for non-string leaves.
- New `i18ncpp_hostile_input_tests` CMake target + `gtest_discover_tests` registration, following the per-file-executable convention established in Phase 2.
- Zero changes to `src/i18ncpp.cpp` — runtime behavior is frozen; this plan is test + fixture only.
</objective>

## AC-1: Four throw sites produce I18NError
```gherkin
Given a default-constructed i18n::I18N instance
When loadLocale is called with any of the following hostile inputs:
  1. a nonexistent file path (tests/fixtures/hostile/does_not_exist.json)
  2. tests/fixtures/hostile/malformed_json.json (unterminated JSON)
  3. tests/fixtures/hostile/not_an_object.json (top-level JSON array)
  4. tests/fixtures/formats_wrong_type.json (array-valued _formats.currency.symbol)
Then loadLocale throws exactly i18n::I18NError
  (asserted via GoogleTest EXPECT_THROW with I18NError as the expected type,
   NOT std::exception or std::runtime_error — the contract is on the concrete type)
```

## AC-2: Non-string leaves are silently skipped (contract documentation)
```gherkin
Given a default-constructed i18n::I18N instance
When loadLocale is called with tests/fixtures/hostile/leaf_is_int.json
  containing {"greeting": 42, "farewell": "bye"}
Then loadLocale does NOT throw (EXPECT_NO_THROW)
  AND tr("greeting") returns "greeting"  // fallback-to-key for absent entries
  AND tr("farewell") returns "bye"        // sibling string entry loaded correctly
```
This documents the project's "skip non-string leaves" contract at the test level
(F-APP-002 open question resolved against src/i18ncpp.cpp:202-228 at apply time).

## AC-3: Post-throw cache invariant holds for formats_wrong_type (P-004 coupling)
```gherkin
Given an i18n::I18N instance with "en" already loaded and both caches warmed
  (tr("greeting") + formatNumber(1234.56) called, ASSERT_GT sizes before throw)
When loadLocale is called with tests/fixtures/formats_wrong_type.json
  and throws I18NError during the configure() staging step
Then formatCacheSize() == 0 AND translationCacheSize() == 0
  (the failed operation must not leave partial side-effects — this couples
   P-005 to the P-004 cache-invariant guarantee)
```

       i18n::I18N inst;
       inst.loadLocale("en", fixturesDir() + "/en.json");
       inst.setLocale("en");

       // Warm both caches (couples P-005 → P-004 post-throw invariant).
       (void)inst.tr("greeting");
       (void)inst.formatNumber(1234.56);
       ASSERT_GT(inst.translationCacheSize(), 0u);

       EXPECT_THROW(

       EXPECT_EQ(inst.formatCacheSize(), 0u);
