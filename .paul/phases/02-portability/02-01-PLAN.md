<objective>
## Goal
Close REVIEW F5 by making the date-formatting path portable across MSVC/GCC/Clang and by pinning an explicit toolchain floor for `std::format`.

## Purpose
The library currently hard-codes `localtime_s` (MSVC-only), so a plain GCC/Clang build fails to compile `formatDateWithConfig`. `std::format` is also used without a documented toolchain floor. Both issues block the "buildable on GCC/Clang" goal of v0.4.2.

## Output
- A portable `localtime` shim used by `formatDateWithConfig`.
- A documented toolchain floor (GCC 13+, Clang 17+, MSVC 19.29+) in README + CLAUDE.md.
- A recorded decision on the `std::format` policy (require floor vs. fallback).
</objective>
