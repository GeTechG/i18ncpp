<objective>
## Goal
Close AEGIS P-004 (cache-invariant parametrized test across all mutators) and P-003 (thread-safety contract docblock + TSan regression test) in a single plan — both target the "invariant by discipline" gap Phase 1 left open.

## Purpose
Phase 1 unified the JSON/filesystem trust boundaries but left two medium findings open:
- F-TEST-002: only 3 of 10 mutators have cache-invalidation coverage; a future commit that forgets `clearFormatCache()` or `clearTranslationCache()` on a new mutator ships silently.
- F-00-002 / F-APP-008 / F-TEST-004: `I18N` has `mutable` caches and scratch buffers written from `const` methods, but no class-level thread-safety docblock. A GTA SA plugin user reading `tr() const` reasonably assumes shared-instance concurrency is safe — and gets an `unordered_map` bucket-corruption segfault instead of stale data.

Both are enforcement gaps around invariants the code currently satisfies — the fix is to make the invariants explicit (test + doc), not to change runtime semantics.

## Output
- Parametrized cache-invariant test covering all 8 public mutators, asserting both `formatCacheSize()` and `translationCacheSize()` return 0 post-mutation.
- Class-level docblock on `I18N` stating "NOT thread-safe, not even for const methods" with the two valid usage patterns.
- Member-level cross-reference on the `mutable` declarations.
- Opt-in TSan regression test gated behind `I18N_ENABLE_TSAN` CMake option, documenting the supported "one instance per thread" pattern.
</objective>

## AC-1: Cache-invariant test covers all 8 mutators
```gherkin
Given an I18N instance with "en" loaded and both caches warmed (tr + formatNumber called)
When any of the following 8 mutators is invoked:
  loadLocale, mergeLocale, load(json), setLocale(string_view),
  setLocale(vector), setFallbackLocale, configure, reset
Then formatCacheSize() == 0 AND translationCacheSize() == 0
  (asserted per-parameter with a failure label naming the mutator)
```

## AC-2: Class-level thread-safety contract is documented
```gherkin
Given a reader opening include/i18ncpp.h
When they reach the `class I18N {` declaration
Then the docblock immediately above the class states the class is NOT thread-safe
  (including for const methods), names the mutable members that violate bitwise
  const, lists the valid usage patterns (one-instance-per-thread OR external mutex),
  and the mutable member declarations carry a cross-reference to the class-level warning
```

  without TSan warnings
And when I18N_ENABLE_TSAN is OFF (the default), the test file is not compiled
  and existing builds on MSVC are unaffected
