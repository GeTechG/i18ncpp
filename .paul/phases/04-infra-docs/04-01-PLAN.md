<objective>
## Goal
Bring the user-facing documentation surface current with v0.4 reality: author a project CLAUDE.md for future AI agents, modernize README.md to reflect C++20 + the new API surface, and refresh BASELINE.md so its build header and recorded numbers match the current codebase.

## Purpose
The v0.4 milestone landed large API and language changes (C++20, `std::span`, simplified overloads, transparent hash, constexpr plural rules, manual parsers) but the docs still describe the v0.2/v0.3 world. README says "C++17 or higher" and lists overloads that no longer exist. BASELINE.md's header says "C++17" and has no v0.4 capture. There is no CLAUDE.md so every future AI session re-derives build/test/layout conventions from scratch. Phase 4 of v0.4.1 exists to close this documentation debt before the next feature cycle.

## Output
- New `CLAUDE.md` at repo root — concise agent onboarding (build, test, layout, conventions, constraints)
- Updated `README.md` — C++20, current API, v0.4 example
- Updated `benchmarks/BASELINE.md` — build header fixed, v0.4 capture section appended (or documented as "no v0.4 hotpath re-measurement taken" if benchmarks were not rerun)
</objective>
