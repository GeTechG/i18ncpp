<objective>
## Goal
Replace nested JSON traversal with flat `unordered_map<string, string>` per locale, achieving O(1) key lookup instead of O(depth) tree walking. Eliminate JSON from runtime storage entirely.

## Purpose
Current `tr()` calls take ~20-170us because each lookup splits the key by dots and traverses the nested JSON tree. Flattening at load time moves this cost to a one-time operation, making every subsequent lookup a single hash map access with pure string values.

## Output
- Modified `include/i18ncpp.h` — new storage type, flatten helper declaration, updated getTranslationData return type
- Modified `src/i18ncpp.cpp` — flatten logic, updated load/merge/lookup/plural/variant methods
</objective>

<task type="auto">
  <name>Task 1: Change storage type and add flatten helper</name>
  <files>include/i18ncpp.h, src/i18ncpp.cpp</files>
  <action>
    **Header (i18ncpp.h):**
    - Change `localesData` type from `std::unordered_map<std::string, json>` to `std::unordered_map<std::string, std::unordered_map<std::string, std::string>>`
    - Change `getTranslationData` return type from `json*`/`const json*` to `const std::string*` (only const version needed now — no mutation of stored data)
    - Remove non-const `getTranslationData` overload (no longer needed — we don't mutate stored translations)
    - Add private method: `void flattenJson(const std::string& prefix, const json& node, std::unordered_map<std::string, std::string>& flatMap);`
    - Remove `dotSplit` declaration

    **Implementation (i18ncpp.cpp):**
    - Implement `flattenJson`:
      - For each key/value in the JSON object (skip `_formats`):
        - Build `fullKey = prefix.empty() ? key : prefix + "." + key`
        - If value is string → `flatMap[fullKey] = value.get<string>()`
        - If value is array → for each element: `flatMap[fullKey + "." + to_string(i)] = element.get<string>()` (string elements only)
        - If value is object AND is a leaf object (all values are strings — plural/variant data) → store each sub-entry: `flatMap[fullKey + "." + subKey] = subValue.get<string>()`
        - If value is object with non-string children → recurse: `flattenJson(fullKey, value, flatMap)`
    - Remove `dotSplit` implementation

    **Helper for leaf object detection:**
    - An object is a "leaf object" if all its values are strings (e.g., `{"one": "...", "other": "..."}`)
    - Objects with nested objects or arrays are containers — recurse into them
  </action>

<task type="auto">
  <name>Task 2: Update load/merge/lookup and plural/variant logic</name>
  <files>src/i18ncpp.cpp</files>
  <action>
    **loadLocale():**
    - After parsing JSON and extracting `_formats`, flatten into `localesData[localeStr]`:
      `localesData[localeStr].clear(); flattenJson("", parsedJson, localesData[localeStr]);`

    **mergeLocale():**
    - Flatten incoming JSON into a temporary flat map
    - Insert/overwrite entries into `localesData[localeStr]` (no json::merge_patch)

    **load():**
    - Iterate top-level keys as locale identifiers
    - For each locale key: extract `_formats`, then flatten its value into `localesData[key]`

    **getTranslationData():**
    - Replace tree traversal with single hash lookup:
      1. Find locale in `localesData` → if not found, return nullptr
      2. Find key in locale's flat map → if not found, return nullptr
      3. Return `&it->second` (const string pointer)

    **tr(key, vector params):**
    - Current: gets json*, checks is_string/is_object, handles plural object
    - New: gets string* from direct lookup
      - If found → interpolateArray and return
      - If not found → try `key + ".other"` as fallback (for when tr() is called on a plural key without count) → if found, interpolateArray and return
      - Otherwise return key as-is

    **trPlural(key, count, params):**
    - Current: gets json*, checks is_object, calls handlePluralWithArray
    - New: compute plural form via `getPluralForm(locale, count)`
      - Look up `key + "." + pluralForm` → if found, interpolateArray
      - Fallback: try `key + ".other"`
      - Fallback: try `key + "." + to_string(count)` (exact count match)
      - Fallback: return key as-is

    **translate(key, json params):**
    - If params has "count": use plural path (build key + "." + pluralForm, look up)
    - Else if direct lookup found: interpolate and return
    - Else: try variant path — iterate param values, try `key + "." + paramValue` as lookup
    - Apply interpolation to final result

    **handlePlural / handlePluralWithArray:**
    - These methods currently take a `const json&` plural object
    - They are no longer needed in their current form — plural logic moves inline into trPlural/translate
    - Remove or refactor: the plural form selection (`getPluralForm`) stays, but the JSON object handling goes away

    **handleVariant:**
    - Similarly no longer takes a JSON object
    - Variant logic moves inline into translate(): iterate params, try key + "." + value

    **keyExists():**
    - Same pattern as getTranslationData — single hash lookup
